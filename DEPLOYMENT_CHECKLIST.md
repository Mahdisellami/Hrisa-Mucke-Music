- [ ] GitHub account

## Phase 1: Backend Deployment to Render

### Step 1: Create Render Web Service

1. Go to [Render Dashboard](https://dashboard.render.com)
2. Click **"New +"** → **"Web Service"**
3. Connect your GitHub repository: `Hrisa-Mucke`
4. Select branch: `main` (or current branch)

### Step 2: Configure Service

Use these settings:

| Setting | Value |
|---------|-------|
| Name | `music-tool-backend` |
| Region | Oregon (or closest to you) |
| Branch | `main` |
| Root Directory | (leave blank) |
| Environment | `Docker` |
| Dockerfile Path | `./backend/Dockerfile.prod` |
| Docker Build Context | `./backend` |
| Plan | **Free** |

### Step 3: Environment Variables

Add these in the **Environment** tab:

#### Required Variables

```bash
# Database (SQLite for free tier)
DATABASE_URL=sqlite:////app/data/music.db

# CORS (will update after Vercel deployment)
ALLOWED_ORIGINS=http://localhost:3000

# JWT (auto-generated by Render)
JWT_SECRET_KEY=<Click "Generate Value">
```

#### Optional Variables (Already set in render.yaml)

```bash
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=15
REFRESH_TOKEN_EXPIRE_DAYS=7
```

### Step 4: Create Database (Optional - PostgreSQL)

**Skip this if using SQLite (free tier recommended)**

If you want persistent PostgreSQL instead:

1. In Render, click **"New +"** → **"PostgreSQL"**
2. Name: `music-tool-db`
3. Plan: **Free** (90-day trial)
4. Copy **Internal Database URL**
5. Update `DATABASE_URL` in backend environment variables

### Step 5: Deploy Backend

1. Click **"Create Web Service"**
2. Wait for build to complete (~5-10 minutes)
3. Deployment logs will show build progress
4. Once deployed, your backend URL will be: `https://music-tool-backend.onrender.com`

### Step 6: Verify Backend

1. Open `https://music-tool-backend.onrender.com` in browser
2. Should see: `{"status":"ok","message":"Music Tool API is running","version":"2.0.0"}`
3. Test API docs: `https://music-tool-backend.onrender.com/docs`

**✅ Backend Deployed!** Note down your backend URL for frontend deployment.

## Phase 2: Frontend Deployment to Vercel

### Step 1: Prepare Frontend

First, verify the frontend build works locally:

```bash
cd mobile-app
npm install
npm run build
```

Expected output: `dist/` folder created successfully

### Step 2: Create Vercel Project

1. Go to [Vercel Dashboard](https://vercel.com/dashboard)
2. Click **"Add New"** → **"Project"**
3. Import your GitHub repository: `Hrisa-Mucke`
4. Select the repository

### Step 3: Configure Build Settings

| Setting | Value |
|---------|-------|
| Framework Preset | **Other** |
| Root Directory | `mobile-app` |
| Build Command | `npm install && npx expo export:web` |
| Output Directory | `dist` |
| Install Command | `npm install` |

### Step 4: Environment Variables

Add in **Settings** → **Environment Variables**:

```bash
EXPO_PUBLIC_API_URL=https://music-tool-backend.onrender.com
```

Replace with your actual backend URL from Phase 1.

### Step 5: Deploy Frontend

1. Click **"Deploy"**
2. Vercel will build and deploy (~3-5 minutes)
3. Your web app will be available at: `https://music-tool.vercel.app` (or similar)

**✅ Frontend Deployed!** Note down your Vercel URL.

## Phase 3: Configure CORS (Critical!)

Your frontend can't communicate with the backend until you update CORS settings.

### Update Backend CORS

1. Go to Render dashboard → Your backend service
2. Go to **Environment** tab
3. Find `ALLOWED_ORIGINS` variable
4. Update value to include your Vercel URL:

```bash
ALLOWED_ORIGINS=https://music-tool.vercel.app,https://music-tool-*.vercel.app
```

**Note:** The `*` wildcard allows preview deployments from Vercel

4. Click **"Save Changes"**
5. Render will automatically redeploy (~2-3 minutes)

### Verify CORS

1. Open your Vercel URL: `https://music-tool.vercel.app`
2. Try to log in or fetch data
3. Check browser console (F12) for CORS errors
4. If you see CORS errors, double-check the ALLOWED_ORIGINS variable

## Phase 4: Initialize Database (Critical!)

Your backend needs test data to function properly.

### Option A: Using Render Shell (Recommended)

1. Go to Render dashboard → Your backend service
2. Click **"Shell"** tab
3. Wait for shell to connect
4. Run initialization commands:

```bash
# Create test data
python scripts/seed_test_data.py

# Classify tracks (if you have tracks)
python scripts/classify_tracks.py
```

### Option B: Using API Endpoints (Alternative)

If shell doesn't work, use the Swagger docs:

1. Go to `https://music-tool-backend.onrender.com/docs`
2. Find `/api/auth/register` endpoint
3. Click **"Try it out"**
4. Register a test user:

```json
{
  "username": "alice",
  "email": "alice@example.com",
  "password": "password123",
  "display_name": "Alice"
}
```

5. Use the registered user to log in on your Vercel frontend

## Phase 5: Verification & Testing

### Test Checklist

- [ ] Backend health check works: `https://your-backend.onrender.com`
- [ ] Frontend loads: `https://your-app.vercel.app`
- [ ] Registration works (create a new user)
- [ ] Login works (get JWT token)
- [ ] Search works (search for tracks - may be empty if no data)
- [ ] Playlists work (create/edit/delete playlist)
- [ ] No CORS errors in browser console

### Common Issues

#### 1. "Failed to fetch" on frontend

**Solution:** Check ALLOWED_ORIGINS in backend environment variables

#### 2. Backend shows "Application failed to respond"

**Solution:** Check Render logs, ensure DATABASE_URL is set correctly

#### 3. Database is empty

**Solution:** Run seed_test_data.py via Render shell

#### 4. Build fails on Vercel

**Solution:** Check build logs, ensure all dependencies in package.json

## Important Notes

### Free Tier Limitations

**Render Free Tier:**
- ⚠️ **No persistent disk** - SQLite database resets on redeploy
- ⚠️ **Spins down after 15 min inactivity** - First request may be slow
- ✅ 750 hours/month (enough for one always-on service)

**Vercel Free Tier:**
- ✅ 100 GB bandwidth/month
- ✅ Unlimited deployments
- ✅ Automatic HTTPS

### Data Persistence

**To avoid data loss on redeployments:**

**Option 1:** Upgrade Render to Starter plan ($7/month)
- Adds persistent disk support
- Add disk via Render dashboard → **Disks** → **Add Disk**
- Mount path: `/app/data`

**Option 2:** Use PostgreSQL database
- Database persists across redeployments
- Free for 90 days, then $7/month

**Option 3:** Accept ephemeral data (development/demo only)
- Re-run seed script after each deployment
