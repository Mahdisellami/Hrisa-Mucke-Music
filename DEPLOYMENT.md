# 🚀 Deployment Guide

This guide covers deploying the Music Tool application using Render (backend) and Vercel (web frontend).

## Architecture

```
┌─────────────────────────────────────────────┐
│                                             │
│  Users                                      │
│    │                                        │
│    ├─► Vercel (Web App)                    │
│    │     └─► music-tool.vercel.app         │
│    │                                        │
│    └─► Render (Backend API)                │
│          └─► music-tool-api.onrender.com   │
│               └─► PostgreSQL Database      │
│               └─► Persistent Disk (MP3s)   │
│                                             │
└─────────────────────────────────────────────┘
```

## Part 1: Deploy Backend to Render

### Step 1: Create Render Web Service

1. Go to [Render Dashboard](https://dashboard.render.com)
2. Click **"New +"** → **"Web Service"**
3. Connect your GitHub repository
4. Configure the service:
   - **Name**: `music-tool-backend`
   - **Region**: Oregon (or closest to you)
   - **Branch**: `main` (or `feature/app`)
   - **Root Directory**: Leave blank
   - **Environment**: `Docker`
   - **Docker Build Context Directory**: `.`
   - **Dockerfile Path**: `./backend/Dockerfile.prod`
   - **Plan**: Free

### Step 2: Add Environment Variables

In Render dashboard, go to **Environment** tab and add:

```bash
# Required
DATABASE_URL=<Your Render PostgreSQL URL or use SQLite>
JWT_SECRET_KEY=<Auto-generated by Render>
ALLOWED_ORIGINS=https://your-app.vercel.app,https://your-custom-domain.com

# Optional
JWT_ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=15
REFRESH_TOKEN_EXPIRE_DAYS=7
```

**For SQLite (Simple, no separate database):**
```bash
DATABASE_URL=sqlite:////app/data/music.db
```

**For PostgreSQL (Recommended for production):**
1. Create a PostgreSQL database in Render
2. Copy the **Internal Database URL**
3. Use it as `DATABASE_URL`

### Step 3: Add Persistent Disk (for MP3 files)

1. In Render dashboard, go to **Disks** tab
2. Click **"Add Disk"**
3. Configure:
   - **Name**: `music-data`
   - **Mount Path**: `/app/data`
   - **Size**: 10 GB (adjust as needed)

### Step 4: Deploy

1. Click **"Create Web Service"**
2. Render will automatically build and deploy
3. Wait for deployment to complete (first build takes ~5-10 minutes)
4. Your backend will be available at: `https://music-tool-backend.onrender.com`

### Step 5: Initialize Database

After first deployment:

```bash
# Option 1: Use Render Shell
# Go to Render dashboard → Shell tab → Run:
python backend/migrations/migrate_to_db.py

# Option 2: Use render.yaml auto-deploy
# The render.yaml file can include initialization scripts
```

## Part 2: Deploy Web App to Vercel

### Step 1: Prepare Expo Web Build

Ensure your `mobile-app/package.json` has web scripts:

```json
{
  "scripts": {
    "web": "expo start --web",
    "build:web": "expo export:web"
  }
}
```

### Step 2: Create Vercel Project

1. Go to [Vercel Dashboard](https://vercel.com/dashboard)
2. Click **"Add New"** → **"Project"**
3. Import your GitHub repository
4. Configure build settings:
   - **Framework Preset**: Other
   - **Root Directory**: `mobile-app`
   - **Build Command**: `npm install && npx expo export:web`
   - **Output Directory**: `dist`
   - **Install Command**: `npm install`

### Step 3: Add Environment Variables

In Vercel dashboard, go to **Settings** → **Environment Variables**:

```bash
EXPO_PUBLIC_API_URL=https://music-tool-backend.onrender.com
```

### Step 4: Deploy

1. Click **"Deploy"**
2. Vercel will build and deploy automatically
3. Your web app will be available at: `https://music-tool.vercel.app`

---

## Part 3: Configure CORS

After both deployments, update backend CORS settings:

1. Go to Render dashboard → Your backend service
2. Update `ALLOWED_ORIGINS` environment variable:

```bash
ALLOWED_ORIGINS=https://music-tool.vercel.app,https://your-custom-domain.com
```

3. Click **"Save Changes"** (triggers redeployment)

## Database Options

### Option A: SQLite (Simplest)

**Pros:**
- No separate database service needed
- Simple setup
- Included in persistent disk

**Cons:**
- Not ideal for high concurrency
- Limited to single instance

**Configuration:**
```bash
DATABASE_URL=sqlite:////app/data/music.db
```

### Option B: Render PostgreSQL (Recommended)

**Pros:**
- Better for production
- Supports multiple connections
- Automatic backups

**Cons:**
- Slightly more complex setup

**Steps:**
1. In Render, create new **PostgreSQL** database
2. Copy **Internal Database URL**
3. Add to backend environment variables
4. Update `DATABASE_URL`

## File Storage Options

### Option A: Render Persistent Disk (Included in guide)

**Pros:**
- Simple, included with Render
- No external dependencies

**Cons:**

## Troubleshooting

### Backend fails to start:
1. Check environment variables are set correctly
2. Verify `DATABASE_URL` format
3. Check Render logs for errors

### Web app can't connect to backend:
1. Verify `EXPO_PUBLIC_API_URL` is set
2. Check CORS settings on backend
3. Ensure backend is deployed and healthy

### Database migration fails:
1. SSH into Render shell
2. Manually run: `python backend/migrations/migrate_to_db.py`
3. Check database permissions

### MP3 files not persisting:
1. Verify persistent disk is mounted at `/app/data`
2. Check disk space usage
3. Ensure file paths in database match disk structure

## Automatic Deployments

Both Render and Vercel support automatic deployments:

Configure in:

- Vercel: **Git** → **Production Branch**
